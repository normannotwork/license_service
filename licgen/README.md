# licgen
Библиотека для работы с лицензией (генерация и проверка).

## Сборка
Осуществляется с помощью CMake. Для сборки также необходимы:
* `gcc`
* `cmake`
* `make`

Перед запуском CMake требуется указать следующие переменные:
* `LICGEN_APPLICATION_NAME` - имя программного продукта, который будет использовать библиотеку
* `LICGEN_APPLICATION_SECRET` - секрет программного продукта, который будет использовать библиотеку - 32 байта в виде hex-строки.

Примечание: секрет удобно генерировать, используя openssl:
```shell
openssl rand -hex 32
```

При использовании IDE (к примеру, CLion), эти переменные указываются в настройках переменных окружения для CMake.

* Примечание: следует задавать разные секреты для сборки тестовых приложений и для использования библиотеки в разработке.

Опции сборки:
* `LICGEN_BUILD_STATIC` - собрать статическую версию библиотеки (по умолчанию стоит сборка динамической библиотеки)
* `LICGEN_BUILD_GENERATOR` - собрать приложение для генерации лицензии
* `LICGEN_BUILD_TESTAPPS` - собрать тестовые приложения
* `LICGEN_HWINFO_USE_RAM` - учитывать обьем установленной оперативной памяти при сборе информации об оборудовании
* `LICGEN_HWINFO_USE_CPU_VENDORS` - учитывать производителя процессора при сборе информации об оборудовании
* `LICGEN_HWINFO_USE_BOARD_VENDOR` - учитывать производителя материнской платы при сборе информации об оборудовании
* `LICGEN_HWINFO_USE_BOARD_NAME` - учитывать модель материнской платы при сборе информации об оборудовании
* `LICGEN_HWINFO_USE_CHASSIS_VENDOR` - учитывать производителя шасси при сборе информации об оборудовании
* `LICGEN_HWINFO_USE_DISK_SERIAL` - учитывать серийный номер жесткого диска, на котором установлена ОС
(для windows - диск С, для Linux - расположение корня файловой системы - `/`)
* `LICGEN_ADD_EXPIRE_TIME` - учитывать время истечения лицензии при генерации лицензии
* `LICGEN_CHECK_VM` - проверка работы на виртуальной машине: если вм обнаружена, то будет сообщено об ошибке
(по умолчанию проверка включена; отключение данной опции рекомендуется только в целях разработки/отладки на виртуальной машине)

## Инструкция по сборке
### Unix
 * Клонировать репозиторий и подмодули:
    * `git clone git@tpts-dev.vniia.net:tius/licgen.git`
 * Создать директорию для сборки и перейти в нее:
    * `mkdir build && cd build`
 * Вызвать cmake с необходимыми опциями:
    * `cmake --build .. --target licgen`

### Windows
 * Клонировать репозиторий и подмодули:
    * `git clone git@tpts-dev.vniia.net:tius/licgen.git`
 * Запустить cmake-gui:
    * Выбрать путь к папке с исходным кодом.
    * Выбрать путь к папке сборки (предварительно создав ее).
    * При необходимости добавить опции сборки с помощью кнопки Add Entry
    * Нажать Configure, выбрать генератор Visual Studio
    * Нажать Generate и Open Project
  * Выполнить сборку в открывшемся окне Visual Studio
   * Из командной строки разработчика Visual Studio в режиме администратора вызвать cmake:
      * `cmake --build {path_to_build_dir} --target licgen`

## Использование
Необходимо подключить `licgen/licgen.h` и добавить библиотеку `licgen` в линковку.

При использовании CMake рекомендуется подключать библиотеку с помощью вызовов `add_subdirectory(licgen)`
и `target_link_libraries([YOUR_APP_TARGET] licgen)`.

## Собираемые приложения
 * `licgen` - библиотека, которую можно использовать в сторонних приложениях
 * `[LICGEN_APPLICATION_NAME]-license-generator` - приложение для генерации лицензии
 * тестовые приложения:
    * `test-hwinfo` - собирает и выводит информацию об оборудовании
    * `test-hash` - вычисляет хеш
    * `test-crypt` - зашифровывает и расшифровывает сообщение
    * `test-licgen` - генерация серийника, генерация и проверка лицензии (без использования файлов для чтения/записи)
    * `test-licgen-file` - генерация серийника, генерация и проверка лицензии (с использованием файлов для чтения/записи)
    * `test-vm` - проверка работы на виртуальной машине
